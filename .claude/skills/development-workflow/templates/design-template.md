## API 设计

### 端点定义

| 方法 | 路径 | 描述 | 请求体 | 响应 |
|------|------|------|--------|------|
| POST | /api/v1/{resource} | 创建 | `{字段说明}` | 201 + 资源对象 |
| GET | /api/v1/{resource}/{id} | 获取 | - | 200 + 资源对象 |
| PUT | /api/v1/{resource}/{id} | 更新 | `{字段说明}` | 200 + 资源对象 |
| DELETE | /api/v1/{resource}/{id} | 删除 | - | 204 |
| GET | /api/v1/{resource} | 列表 | 查询参数 | 200 + 分页列表 |

### 错误响应

| HTTP 状态码 | 场景 | 错误消息 |
|-------------|------|----------|
| 400 | 输入验证失败 | 详细验证错误 |
| 401 | 未认证 | Invalid credentials |
| 403 | 权限不足 | Not authorized |
| 404 | 资源不存在 | Resource not found |
| 409 | 资源冲突 | Resource already exists |

## 核心逻辑

描述关键业务逻辑的处理流程，用伪代码或流程说明：

```
创建资源：
  1. 验证输入数据
```
