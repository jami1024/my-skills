# 架构评审清单

## 使用说明

在以下场景使用此清单：
- 新项目启动前的架构设计评审
- 重大功能开发前的设计评审
- 定期架构审查

**评分**：✅ 符合 | ⚠️ 需改进 | ❌ 不符合 | N/A 不适用

---

## 1. 架构原则

### SOLID

- [ ] **单一职责**：每个模块/类只负责一件事
- [ ] **开放封闭**：对扩展开放，对修改封闭
- [ ] **里氏替换**：子类/实现可以替换父类/接口
- [ ] **接口隔离**：接口小而专注，避免臃肿
- [ ] **依赖倒置**：依赖抽象而非具体实现

### 分层

- [ ] **层次清晰**：展示层、业务层、数据层分明
- [ ] **依赖方向正确**：上层依赖下层，不反向
- [ ] **无跨层调用**：不跳过中间层直接调用

---

## 2. 项目结构

- [ ] **按领域/功能组织**：而非按技术层级
- [ ] **模块化**：每个功能模块相对独立
- [ ] **配置集中**：配置文件统一管理，环境变量分离
- [ ] **命名一致**：遵循统一的命名规范

---

## 3. API 设计

- [ ] **资源命名**：使用名词，而非动词
- [ ] **HTTP 方法正确**：GET 查询、POST 创建、PUT/PATCH 更新、DELETE 删除
- [ ] **状态码准确**：200/201/204/400/401/403/404/409/500 使用正确
- [ ] **版本化**：API 有版本管理策略
- [ ] **文档完整**：每个端点有描述、请求/响应示例

---

## 4. 数据层

### 数据库设计

- [ ] **命名规范**：表名、字段名遵循统一规范
- [ ] **主键设计**：每个表有合理主键
- [ ] **外键约束**：关联关系用外键表达
- [ ] **索引合理**：WHERE/JOIN/ORDER BY 字段有索引，无过度索引
- [ ] **时间戳**：created_at、updated_at 字段

### 数据访问

- [ ] **避免 N+1 查询**：关联查询使用预加载
- [ ] **事务边界清晰**：在业务层管理事务
- [ ] **错误回滚**：异常时自动回滚
- [ ] **幂等性**：关键操作支持幂等

---

## 5. 业务逻辑

- [ ] **职责单一**：每个业务模块负责一个领域
- [ ] **方法命名清晰**：动词+名词，表意明确
- [ ] **依赖注入**：通过参数或构造函数注入依赖
- [ ] **业务验证**：复杂规则在业务层验证
- [ ] **避免重复**：公共逻辑提取复用

### 错误处理

- [ ] **分层异常**：领域异常 → 应用异常 → HTTP 响应
- [ ] **异常信息完整**：包含上下文（ID、操作、原因）
- [ ] **全局异常处理**：统一的异常处理器

---

## 6. 认证和授权

### 认证

- [ ] **认证方式合理**：JWT / Session / OAuth2 选择有理由
- [ ] **Token 安全**：合理过期时间、刷新机制
- [ ] **密码安全**：安全哈希算法，永不明文存储

### 授权

- [ ] **权限模型清晰**：RBAC 或 ABAC
- [ ] **每个受保护端点都有权限检查**
- [ ] **最小权限原则**

---

## 7. 性能

- [ ] **查询优化**：索引、预加载、避免全表扫描
- [ ] **分页实现**：大数据量使用分页（offset 或游标）
- [ ] **缓存策略**：热点数据缓存，有失效机制
- [ ] **异步处理**：I/O 密集操作使用异步
- [ ] **批量操作**：批量插入/更新而非循环

---

## 8. 安全性

- [ ] **输入验证**：所有外部输入都验证，白名单优先
- [ ] **注入防护**：SQL 注入、XSS、CSRF
- [ ] **HTTPS**：生产环境强制 HTTPS
- [ ] **CORS 配置**：限制允许的域名
- [ ] **速率限制**：防暴力破解和滥用
- [ ] **敏感信息**：不记录到日志，不返回给客户端，使用环境变量存储密钥

---

## 9. 可维护性

### 代码质量

- [ ] **格式统一**：使用格式化工具
- [ ] **类型标注**：函数签名有类型信息
- [ ] **魔法值消除**：使用常量或枚举

### 测试

- [ ] **覆盖率达标**：> 80%
- [ ] **单元测试**：业务逻辑层
- [ ] **集成测试**：API 端点
- [ ] **测试命名清晰**：`test_{功能}_{场景}_{预期结果}`

### 日志

- [ ] **结构化日志**：JSON 格式或统一格式
- [ ] **级别正确**：DEBUG/INFO/WARN/ERROR 分级合理
- [ ] **含上下文**：request_id、user_id 等关联信息
- [ ] **敏感信息脱敏**

---

## 10. 可扩展性

- [ ] **无状态设计**：不依赖本地状态，支持多实例
- [ ] **连接池配置合理**：数据库、缓存连接池
- [ ] **健康检查端点**：支持负载均衡器探测
- [ ] **监控指标**：请求量、响应时间、错误率
- [ ] **错误追踪**：集成错误监控服务

---

## 11. 部署和运维

- [ ] **环境分离**：dev / staging / prod 配置分离
- [ ] **敏感配置不入库**：.env 或密钥管理服务
- [ ] **配置启动验证**：缺少必需配置时快速失败
- [ ] **容器化**：Dockerfile 优化（多阶段构建、镜像精简）
- [ ] **CI/CD**：自动化测试、代码检查、部署
